# ねこつなげる - 開発計画

## 技術構成
- HTML + JavaScript + Canvas（フレームワークなし）
- 単一HTMLファイル or 小規模なJS分割
- 素材: PNG画像（自作、MSペイント風白黒）
- 公開: 静的サイトとしてitch.io or レンタルサーバー

## フェーズ1: 最小動作版（MVP）

### 1-1. プロジェクト骨格 ✅ 完了
- [x] index.html + main.js + style.css の作成
- [x] Canvas描画の初期化（9:16比率、レスポンシブ対応）
- [x] ゲームループ（requestAnimationFrame）
- [x] PC余白レイアウト（中央にゲーム画面、左右に情報エリア）

#### 1-1 実装メモ
- Canvas は `resizeCanvas()` でウィンドウサイズに追従し、9:16比率を維持
- スマホ: 幅基準で最大化、PC: 高さ基準で中央配置
- サイドパネル（SCORE/NEXT）は `@media (min-aspect-ratio: 10/16)` で横長画面のみ表示
- ゲームループは `requestAnimationFrame` + dt（秒単位）で駆動
- プレースホルダーとして8×16グリッド線を描画済み
- デバッグ用にCanvas解像度を左下に表示

### 1-2. 盤面 ✅ 完了
- [x] 8×16の2次元配列で盤面管理
- [x] グリッド描画（プレースホルダー: 色付き四角形）
- [x] ブロックデータ構造の定義
  - 各ブロック: { type, sides: [上,右,下,左], rotation }
  - sides の各値: 0〜4

#### 1-2 実装メモ
- `createBoard()` で 8×16 の2次元配列を生成。`board[row][col]` で row=0 が最上段
- 各マスは `null`（空）または ブロックオブジェクト `{ type, sides, rotation }`
- `createBlock(sides, type?)` でブロック生成。type を省略すると `classifyBlock()` で自動分類
- 自動分類は接続面数で判定: 0=isolated, 1=end(頭/おしり), 2=straight/elbow, 3=tee, 4=cross
- `rotateSides(sides)` で時計回り90°回転: `[上,右,下,左]` → `[左,上,右,下]`
- 45パターンのブロック定義を `BLOCK_PATTERNS` 配列に格納済み
- 描画: タイプ別に色分け＋接続面の方向マーカー＋辺の値をデバッグ表示
- テスト用に `placeTestBlocks()` で各タイプのサンプルブロックを配置して動作確認可能

### 1-3. ブロック定義 ✅ 完了
- [x] 45パターンのブロックデータをJSONまたはJS定数で定義
- [x] 回転処理（sides配列を回転: [上,右,下,左] → [左,上,右,下]）
- [x] パーツカテゴリ分類（頭/直線/L字/T字/十字/おしり）
- [x] 各カテゴリの出現確率テーブル

#### 1-3 実装メモ
- `BLOCK_CATEGORY_INDICES` で45パターンをカテゴリ別インデックスに分類
  - isolated:1, end:4, elbow:8, straight:10, tee:16, cross:6 = 計45
- head/tail は共通の end パターン（4種）を使い、type フィールドで区別
- `SPAWN_WEIGHTS` で出現確率を定義（重み合計80、爆弾/パック未実装分は除外）
  - head:10, tail:10, straight:15, elbow:25, tee:15, cross:5
- `generateBlock()` でカテゴリ抽選 → パターン選択 → ランダム回転(0〜3回)
- `TYPE_COLORS` に head/tail を個別色で追加（head:黄色, tail:オレンジ）
- **発見**: T字は SPEC では0パターンと記載されているが、実際の45パターン中に16個存在する。
  出現確率テーブルにはT字を含めて実装済み（PLAN.md 未決定事項1 に関連）

### 1-4. ブロック落下 ✅ 完了
- [x] 新規ブロック生成（確率テーブルに基づくランダム選択）
- [x] 一定間隔で1マス落下
- [x] 着地判定（床 or 他ブロックに接触で固定）
- [x] ネクスト表示（次の2個）

#### 1-4 実装メモ
- `spawnBlock()` でネクストキューから取り出し → 新しいブロックをキュー末尾に追加
- 出現位置: col=3, row=0（盤面中央付近の最上段）
- 通常落下: 1.0秒/マス、高速落下: 0.05秒/マス
- ネクスト表示: Canvas右上にオーバーレイ描画（背景付き、2個表示）
- ゴースト表示: 着地予測位置を半透明で描画（プレイアビリティ向上）

### 1-5. 操作 ✅ 完了
- [x] PC: 矢印キーで左右移動、上キーで回転、下キーで高速落下
- [x] スマホ: スワイプで左右、タップで回転、長押しで高速落下
- [x] 操作のデバウンス/スロットリング

#### 1-5 実装メモ
- PC: ArrowLeft/Right=移動、ArrowUp/Space=回転、ArrowDown(押し続け)=高速落下
- スマホ: touchmove でセル幅60%以上のスワイプで移動（連続スワイプ対応）
- タップ判定: 250ms以内かつ移動20px以内 → 回転
- 長押し判定: 300ms以上ホールド → 高速落下開始、指を離すと解除
- `e.preventDefault()` でスクロール等のデフォルト動作を抑制

### 1-6. 完成判定 ✅ 完了
- [x] 頭ブロック一覧の取得
- [x] 各頭からBFS/DFSで接続探索
  - 隣接マスの辺の値を照合
  - 禁止ペアでないことを確認
  - おしりに到達したら完成
- [x] 完成した猫のブロックを盤面から消去
- [x] 消去後の落下処理（上のブロックが落ちる）
- [x] 落下後に再度完成判定（連鎖チェック）

#### 1-6 実装メモ
- `FORBIDDEN_PAIRS` Set で禁止ペアを高速判定
- `DIRECTIONS` 配列で4方向の探索を統一的に処理（dRow, dCol, 自辺index, 隣辺index）
- `findCompletedCats()`: 全headブロックからBFS → tailに到達すれば完成
- 頭だけの猫: 接続面が全て壁or非互換ブロックに面している場合のみ完成扱い
  （空きマスに面している場合はまだ繋がる可能性ありとして温存）
- `applyGravity()`: 列ごとに下から詰めるアルゴリズム
- 連鎖: `processCompletions()` 再帰呼び出しで実現
- スコア: ブロック数 × 10点/ブロック

### 1-7. ゲームオーバー ✅ 完了
- [x] 出現位置にブロックがある場合の検知
- [x] ゲームオーバー画面表示

#### 1-7 実装メモ
- `spawnBlock()` 内で `canPlaceAt(SPAWN_ROW, SPAWN_COL)` を確認
- 配置不可なら `game.state = "gameover"` に遷移
- 画面: 半透明黒オーバーレイ + 「ねこづまり！」テキスト + スコア表示
- リスタート: スペース/Enter/タップで `restartGame()` を呼び出し
- リスタート時に全状態をリセット（盤面・スコア・キュー等）
