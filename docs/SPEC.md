# ねこつなげる - ゲーム仕様書

## 概要
落ちてくるブロック（猫パーツ）を操作し、頭→胴体→おしりを繋げて猫を完成させる落ち物パズルゲーム。
完成した猫はコレクションされ、変な形の猫を眺めて笑うのが目的。

## コンセプト
- **ジャンル**: 落ち物パズル（テトリス × 水道管ゲーム × ネココレクション）
- **方向性**: コレクション寄り。難しさよりも「出来上がった猫の形で笑う」体験を重視
- **雰囲気**: シュール・のんびり・かわいい（バカゲーではなくシュールゆる系）
- **テンポ**: テトリスと同程度の落下速度。演出だけのんびりかわいく

## 技術スタック
- **開発**: HTML + JavaScript + Canvas
- **公開先**: itch.io（無料公開）、または自前レンタルサーバー
- **対応端末**: スマホメイン、PC副次対応

## 画面
- **基本比率**: 9:16（スマホ縦持ち）
- **PC表示時**: 中央にゲーム画面、左右余白にスコア等を表示
- **ビジュアル**: MSペイントで雑に描いた風の白黒線画＋塗り
  - ふんわり手書き・低解像度
  - 線のムラやガタつきが味になる

## ブロック仕様

### 基本ルール
- 全て1マス単位で構成
- 各ブロックは4辺（上・右・下・左）に接続情報を持つ
- 落ちてくる順番は完全ランダム
- 落下中に左右移動・回転が可能

### 辺の値の定義
各辺は0〜4の値を取る。接続面を右に置いたときの定義:

| 値 | 意味 |
|----|------|
| 0 | 接続なし |
| 1 | 上下足なし |
| 2 | 上が足なし、下が足あり |
| 3 | 上が足あり、下が足なし |
| 4 | 上下足あり |

### 接続ルール（禁止ペア）
隣接する2辺が以下の並びになる場合、接続不可:
- 3→4, 4→3（背と腹が繋がる）
- 1→1, 2→2（同種の重複）
- 1→3, 2→4, 4→1, 3→2（追加禁止ペア）

背と腹の接続は許可しない（腹/腹、背/背のみ接続可能）。

### パーツ種類と回転考慮後のパターン数
回転で同じになるパターンを除外し、禁止ペアも除外した結果: **45通り**

| パーツ | パターン数（回転除外前） | 備考 |
|--------|------------------------|------|
| 頭 | 2 | 背中向き / 腹向き |
| 直線 | 1 | 回転すれば同じ |
| L字 | 2 | 回転で4方向だが背腹で2パターン |
| T字 | 0 | 背/腹制約で存在しない |
| 十字（4方向） | 1 | 回転しても同じ |
| おしり | 2 | 背中向き / 腹向き |

※上記は背/腹の組み合わせを考慮した数。全45パターンの列挙はGPTチャットに記載あり。

## 完成判定アルゴリズム

### 基本方針
- 常に「頭一覧」から探索する方式で統一（シンプルさ優先）
- 盤面サイズが小さいので計算量は問題にならない

### 処理フロー
1. 盤面上の全ブロックから連結成分（BFS）を探索
2. 各連結成分について、全ブロックの非ゼロ接続面をチェック
3. 全ての非ゼロ接続面が空きマスにも壁（盤面外）にも面していなければ「完成」
4. 完成した猫のブロックを盤面から消去
5. 猫の形状データをギャラリーログに保存

### 完成条件の詳細
- 接続面が空きマスに面している → 未完成（まだ繋がる可能性あり）
- 接続面が壁（盤面外）に面している → 未完成（死にブロック扱い）
- 接続面が他ブロックに面している（互換・非互換問わず） → 塞がれている
- 全接続面が塞がれている → 完成

### 同時完成の扱い
- 複数の猫が同時に完成した場合、内部処理は一括消去
- 演出は1匹ずつ順番にズームアップ表示
- 同時消しは連鎖とはみなさない（ぷよぷよ準拠）

### 連鎖
- 消去→落下→再度消去が発生した場合が連鎖

### 特殊な完成パターン
- 頭だけの猫: 許可（接続面が全て塞がれていれば完成）
- おしりだけの猫: 許可（同上）
- ループ状の猫（おしりなし）: 許可（全接続が閉じていれば完成）
- 胴体だけのループ: 許可（頭もおしりもなくても完成）

## 死にブロック判定

### 分類
| 種類 | 説明 | 灰色表示 |
|------|------|----------|
| 完全死 | 壁に接続面が塞がれている。永久に復活しない | する |
| 伝播死 | 壁に接触していないが、落下先も含め全経路が塞がれている | する（理想）|
| 仮死 | 今は塞がれているが将来復活の可能性あり | しない |

### 判定の難しさ
- 壁接触 = 完全死（確定。下のブロックが消えても壁接触は変わらない）
- 壁に接触していないブロックでも、落下シミュレーションして全経路が死んでいれば完全死
- ただし「伝播死」の正確な判定にはシミュレーションが必要で計算コストが高い
- 実装優先度: まずは壁接触の完全死のみ灰色表示 → 余裕があれば伝播死も対応

### 判定タイミング
- ブロック落下・固定後
- ブロック消去後
- 毎回盤面全体を再評価

## お助けアイテム（拡張機能・今後の展望）

### ネコ爆弾
- 効果: 上下左右1マスのブロックを消去（中心は含まない）
- 落下ブロックとして登場
- 出現率: 低め（4〜5%程度）

### 今後追加候補

- 縦一列消去
- 万能接続ブロック（どの方向でも繋がる）

## 操作方法

### スマホ（メイン）
| 操作 | アクション |
|------|-----------|
| 左右スワイプ | ブロック左右移動 |
| タップ | ブロック回転 |
| 長押し | 高速落下 |

### PC（副次）
| 操作 | アクション |
|------|-----------|
| ←→キー | ブロック左右移動 |
| ↑ or スペース | ブロック回転 |
| ↓キー | 高速落下 |

## ゲーム終了条件
- 新しいブロックの出現位置にすでにブロックがある場合 → ゲームオーバー
- 「ねこづまり！」的なコミカルメッセージを表示

## ギャラリー（コレクション）

### 保存方式
- 猫の形状データ（構成ブロックID + 配置座標 + 回転情報）をバイナリ/JSONで保存
- 画像データは保存しない。表示時にブロック画像を当てはめて再構成
- 全ての完成猫を自動保存（上限なし）

### 表示
- 完成した猫がぽよぽよ動く（微妙な呼吸アニメ、尻尾揺れ程度）
- 喋らない
- タップすると「ぷにっ」と体がへこむ → 戻る

### 名前・コメント
- ユーザーが自由にテキスト入力して名前やコメントをつけられる
- 自動名前生成は将来対応（辞書組み合わせ方式: 前半＋後半で「ぐにゃたま」等）

---
